# 多项目AI工具状态管理器需求整理

## 项目背景
个人使用的多项目管理工具，主要解决AI开发工具执行完成后无法获知状态的问题。每个项目的AI工具执行状态需要独立管理。

## 核心需求

### 1. 项目管理
#### 1.1 项目列表
- [ ] 项目创建、编辑、删除
- [ ] 项目状态标记（活跃/隐藏）
- [ ] 项目基本信息（名称、Cursor 密钥、UUID自动生成、AI执行状态）
- [ ] 所有项目、任务，在一个页面显示。显示待执行任务总个数。可以展开任务。默认只显示5个任务。

#### 1.2 任务管理
- [ ] 任务创建、编辑、删除（文本描述）
- [ ] 任务状态管理（待处理/进行中/已完成）
- [ ] 任务优先级设置（拖拽设置优先级、置顶）
- [ ] 任务与项目关联（每个任务属于特定项目）
- [ ] 任务列表展示（按项目分组，按优先级排序）

#### 1.3 项目关联
- [ ] 每个项目独立管理AI工具执行状态
- [ ] 项目间AI执行状态隔离
- [ ] 项目切换时显示对应AI执行历史和任务列表

### 2. AI工具状态监控
#### 2.1 执行状态跟踪
- [ ] 实时监控AI工具执行状态（通过WebSocket/轮询）
- [ ] 执行进度可视化（进度条/状态指示器）
- [ ] 执行结果状态显示（运行中/成功/失败/警告）
- [ ] 执行时间显示（开始时间/耗时）
- [ ] 支持手动状态更新（当自动监控不可用时）

#### 2.2 AI工具集成方案
- [ ] 本地代理服务（可选）：监听本地AI工具执行状态
- [ ] IDE插件集成（推荐）：Cursor/VS Code插件自动上报状态
- [ ] 手动状态同步：用户主动更新执行状态
- [ ] API接口：第三方AI工具通过API推送状态更新

#### 2.3 项目关联状态
- [ ] 每个项目的AI执行状态独立显示
- [ ] 当前项目的执行状态突出显示
- [ ] 项目切换时自动切换状态视图
- [ ] 多项目同时执行时的状态管理

#### 2.3 通知机制
- [ ] 浏览器通知API（需要用户授权）
- [ ] 页面内Toast通知（执行完成时自动消失，无需手动关闭）
- [ ] 声音提示（可选）
- [ ] 实时状态更新（WebSocket推送）
- [ ] 通知内容包含项目名称和执行结果

#### 2.4 历史记录
- [ ] 每个项目的AI执行历史
- [ ] 执行成功率统计（按项目）
- [ ] 失败原因记录
- [ ] 历史记录清理功能

## 技术实现方案

### 架构设计
- **前端应用**: Vue.js 3 + TypeScript 单页面应用
- **后端服务**: Node.js + Express API服务
- **数据库**: PostgreSQL
- **通信方式**: WebSocket（实时状态更新）
- **部署方式**: 传统部署方式

### 技术栈详情
- **前端框架**: Vue.js 3 + TypeScript + Vite
- **状态管理**: Pinia
- **UI组件**: Element Plus
- **HTTP客户端**: Axios
- **路由**: Vue Router 4
- **后端框架**: Node.js + Express + Socket.io
- **数据库ORM**: Prisma
- **身份验证**: JWT + Passport.js

### 核心功能模块
1. **用户认证模块**: JWT身份验证
2. **项目管理模块**: 项目CRUD操作
3. **AI监控模块**: 执行状态跟踪和通知
4. **数据存储模块**: 数据库操作
5. **实时通信模块**: WebSocket状态推送
